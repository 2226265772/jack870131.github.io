算法 - 排序(一)

時間複雜度

  • 常數時間的操作: 一個操作如果和數據量沒有關係，每次都是固定時間內完成的操作，叫做常數操作。
  • 時間複雜度為一個算法流程中，最差情況下，常數操作數量的指標。常用 O (Big O) 來表示。具體來說，在常數操作數量的表達式中，只要高階項，不要低階項，也不要高階項的係數，剩下的部分如果記為 f(N), 則時間複雜度為 O(f(N))
  • 評價一個算法流程的好壞，先看時間複雜度的指標，然後再分析不同數據樣本下的實際運行時間，也就是常數項時間
  • 比較算法效率: 先看指標(最高位)，再看係數(最高位係數)

Example

  • Q: 一有序數組A，另一無序数組B，打印B中所有不在A中的數，A數組長度為N，B數組長度為M

    • 算法1: 對於B中的每一個數，都在A中通過遍歷的方式查找
    • 算法2: 對於B中的每一個數，都在A中通過二分的方式查找
    • 算法3: 先把數組B排序，然後用類似外排的方式打印所有再A中出現的數

    分析三個算法的時間複雜度並分析好壞

  • Ans:

    • 算法1:

      通過算法1我們可以發現B每循環一個數，A都要查找N次。由此可知算法複雜度為: O($M * N$)

    • 算法2:

      先分析二分查找的複雜度。舉例來說，如果有8個數，要想查找出某一個數，二分查找共需要經過3次運算方可得出結果。由此可知二分查找的複雜度為O($log_2N$)

      由於需循環B中的每一個數，所以與二分查找的複雜度相結合其複雜度為: O($M * logN$)

    • 算法3:

      先假設排序B所需的時間複雜度為 O($M*logM$)

      比較A數組與B數組(現B已為有序)，只要A中的數小於B中的數則A數組往後一位查找，如果相等則不打印並且B往後一位查找。直到A大於B中的數。如果此時仍未查找到到B中的數則打印並且B往後一位查找。

      由此可知，A數組最多指向N個數，B數組最多指向N個數。只要其中一個數組結束查找(到達盡頭)則整個運算結束。時間複雜度為 O($N + M$)，與先前的排序相結合，時間複雜度為: O($M * logM$) + O($N + M$)

對數器

  • 有一個你想要測的方法a
  • 實現一個絕對正確但是複雜度不好的方法b
  • 實現一個隨機樣本產生器
  • 實現比對的方法
  • 把方法a和方法b比對很多次來驗證方法a是否正確
  • 如果有一個樣本使得比對出錯，打印樣本分析是哪個方法出錯
  • 當樣本數量很多時，比對測試依然正確，可以確定方法a已經為正確的

冒泡排序

  • 冒泡排序的方法:

    每循環到一個數就與其後面一個數比較(循環到第1位則第1位和第2位比較，循環到第二位則第2位和第3位比較)，如果前一個數比後一個數大則進行交換，反之則不交換。

    循環完一輪後，這輪的最後一個數都會是最大的，下一次循環不須要循環到上一輪的最後一個數。

  • 時間複雜度: 第一次循環n個數，第二次循環n - 1個數，以此類推。最終形成一等差數列 $aN^2 + bN + 1$

    時間複雜度: O($n^2$)

    空間複雜度: O(1)

package pers.sort;

import java.util.Arrays;

/**
 * 冒泡排序
 *
 * @author jack870131
 */
public class BubbleSort {
    public static void bubbleSort(int[] arr) {
        //數組為空或長度小於2不需要排序時直接返回
        if (arr == null || arr.length < 2) {
            return;
        }
        //每一回合最後一個數都是最大的，故下次回合不需要再對上一會核最後一個數進行比較
        for (int end = arr.length - 1; end > 0; end--) {
            for (int i = 0; i < end; i++) {
                //每循環到一個數便與後面一位進行比較
                if (arr[i] > arr[i + 1]) {
                    swap(arr, i, i + 1);
                }
            }
        }
    }

    public static void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}

選擇排序

  • 選擇排序的方法:

    0到 N - 1位的範圍上找最小的數擺在第1位，0到 N - 2 位的範圍上找最小的數擺在第2位，以此類推。

  • 時間複雜度: O($n^2$)

    空間複雜度: O(1)

package pers.sort;

/**
 * 選擇排序
 * @author jack870131
 */
public class SelectionSort {
    public static void selectionSort(int[] arr) {
        //如果数組為空或長度小於2不需要進行比較時直接返回
        if(arr == null || arr.length < 2) {
            return;
        }
         //每一次循環到一個數都與其後面的所有數進行比較
        for(int i = 0; i < arr.length - 1; i++) {
            int minIndex = i;
            //找到其後最小的數的下標
            for(int j = i + 1; j < arr.length; j++) {
                minIndex = arr[j] < arr[minIndex] ? j : minIndex;
            }
            //進行交換
            swap(arr, i, minIndex);
        }
    }

    public static void swap(int[] arr, int i, int minIndex) {
        int temp = arr[i];
        arr[i] = arr[minIndex];
        arr[minIndex] = temp;
    }
}

插入排序

  • 插入排序的方法:

    就像玩牌時一樣，一次抓一堆牌進行排列。由左往右每次抓一張牌，每抓起一張則檢查是否可以將其插入左邊的牌中。也就是每一次循環都使0至循環到的數之前的每一位都排序好。

  • 時間複雜度:

    • 最差情況(完全倒序): O($N^2$)
    • 最好情況(完全順序): O($N$)
    • 空間複雜度 O(1)
package pers.sort;

/**
 * 選擇排序
 *
 * @author jack870131
 */
public class InsertionSort {
    public static void insertionSort(int[] arr) {
        //如果数組為空或長度小於2則直接返回
        if (arr == null || arr.length < 2) {
            return;
        }
        //每次循環一個數
        for (int i = 1; i < arr.length; i++) {
            /*
             * 循環到的數的前一位數與其進行比較，如果大於則進行交換，
             * 不斷往前進行比較直到循環到的數之前的數都排序好才進行
             * 下一次循環
             */
            for (int j = i - 1; j >= 0 && arr[j] > arr[j + 1]; j--) {
                swap(arr, j, j + 1);
            }
        }
    }

    public static void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}

遞歸

剖析遞歸行為和遞歸行為時間複雜度的估算

master 公式的使用

n: 主問題的樣本量

a: 該過程一共發生多少次

n / b: 被拆成子問題後的樣本量

O($n ^ d$): 除去子過程調用之外，剩下過程的時間複雜度

$T(n) = a * T (n / b) + O(n ^ d)$

  1. log(b, a) > d ⇒ 複雜度為 $O(N ^{log(b, a)})$
  2. log(b, a) = d ⇒ 複雜度為 $O({N^d} * {logN}$)
  3. log(b, a) < d ⇒ 複雜度為 $O(N^d)$

歸併排序

  • 將一數組對半拆分為子數組

    創建一輔助數組，比較左右兩子數組中的數，依順序填入輔助数組中

    待輔助數組填鴨完成後再拷貝回原数組中完成排序

  • 歸併排序複雜度

    時間複雜度:

    $T(n) = 2 T (n / 2) + O(n)$

    ⇒ O(N + logN)

    空間複雜度:

    ⇒ O(N)

package pers.sort;

/**
 * 歸併排序
 * @author jack870131
 */
public class MergeSort {
    public static void mergeSort(int[] arr) {
        if(arr == null || arr.length < 2) {
            return;
        }
        sortProcess(arr, 0, arr.length - 1);
    }

    public static void sortProcess(int[] arr, int left, int right) {
        if(left == right) {
            return;
        }
        int mid = left + ((right - left) >> 1); //left 和 right 中點的位置 (left + right) / 2
        //左部分排序  T(N / 2)
        sortProcess(arr, left, mid);
        //右部分排序 T(N / 2)
        sortProcess(arr, mid + 1, right);
        //合併數組  O(N)
        merge(arr, left, mid, right);
        // T(N) = 2 T(N / 2) + O(N)
    }

    public static void merge(int[] arr, int left, int mid, int right) {
        //輔助數組
        int[] help = new int[right - left + 1];
        int i = 0;
        int p1 = left;
        int p2 = mid + 1;
        while(p1 <= mid && p2 <= right) {
            help[i++] = arr[p1] < arr[p2] ? arr[p1++] : arr[p2++];
        }
        //兩個必有一個越界
        while(p1 <= mid) {
            help[i++] = arr[p1++];
        }
        while(p2 <= right) {
            help[i++] = arr[p2++];
        }
        for(i = 0; i < help.length; i++) {
            arr[left + i] = help[i];
        }
    }
}

小和問題

在一個數組中，每一個數左邊比當前數小的數累加起來，叫做這個數組的小和。求一個數組
的小和。
例子：
[1,3,4,2,5]
1左邊比1小的數，沒有；
3左邊比3小的數，1；
4左邊比4小的數，1、3；
2左邊比2小的數，1；
5左邊比5小的數，1、3、4、2；
所以小和為1+1+3+1+1+3+4+2=16

package pers.sort;

/**
 * 小和問題
 * @author jack870131
 */
public class SmallSum {
    public static int smallSum(int[] arr) {
        if(arr == null || arr.length < 2) {
            return 0;
        }
        return mergeSort(arr, 0, arr.length - 1);
    }

    public static int mergeSort(int[] arr, int l, int r) {
        if(l == r) {
            return 0;
        }
        int mid = l + ((r - l) >> 1); //mid = left + (right - left) / 2
        return mergeSort(arr, 1, mid)
                + mergeSort(arr, mid + 1, r)
                + merge(arr, 1, mid, r);
    }

    public static int merge(int[] arr, int l, int m, int r) {
        int[] help = new int[r - 1 + 1];
        int i = 0;
        int p1 = 1;
        int p2 = m + 1;
        int res = 0;
        while(p1 <= m && p2 <= r) {
            //p1 比 p2 小時，p2後面有多少個數乘當前p1自己 = p1 被榨出來的數量
            res += arr[p1] < arr[p2] ? (r - p2 + 1) * arr[p1] : 0;
            help[i++] = arr[p1] < arr[p2] ? arr[p1++] : arr[p2++];
        }
        while(p1 <= m) {
            help[i++] = arr[p1++];
        }
        while(p2 <= r) {
            help[i++] = arr[p2++];
        }
        for(i = 0; i < help.length; i++) {
            arr[l + i] = help[i];
        }
        return res;
    }
}
