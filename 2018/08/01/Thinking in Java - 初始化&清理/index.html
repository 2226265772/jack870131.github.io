Thinking in Java - 初始化&清理

Constructor 構造器

  • 在 Java 中，”初始化“與”創建“綑綁在一起

  • 為了讓方法名相同而形式參數不同的構造器同時存在，必須用到方法重載

  • 每個重載的方法都必須有一個獨一無二的參數類型列表(有相同的參數但順序不同也是可以的)

  • 如果寫的類中沒有構造器，則編譯器會自動創建一個默認的無參構造器

    如果已經寫了一個構造器(不管有無參數)，編譯器便不會再自動創建無參構造器(如果需要，需手動創建)

  • 構造器沒有顯示的使用 static 關鍵字，但實際上也是靜態方法

this 關鍵字

  • this 關鍵字只能再方法內部使用，表示對”調用方法的那個對象“的引用

    通常寫 this 的時候都是指”這個對象“或”當前對象“，而他本身表示對當前對象的引用

  • 如果在方法內部調用同一個類的另一個方法，就不必使用 this, 直接調用即可

  • 如果 this 添加了參數列表(如: this(xxx, xxx)), 將產生對符合此參數列表的某個構造器的明確調用

重新認識 static

了解 this 關鍵字後，能夠更加全面的認識 static (靜態) 方法的涵義。

  • static 方法是沒有 this 的方法

  • 在 static 方法的內部不能調用非靜態方法，反過來是可以的

    可以在沒有創建任何對象的前提下，僅僅通過類本身來調用 static 方法

清理: 終結處理和垃圾回收

  • 垃圾回收要點:

    • 對象可能不被垃圾回收
    • 垃圾回收並不等於 C++ 的”析構”(C++ 中銷毀對象必須用到這個函數
    • 垃圾回收只與內存有關

  • Java 的垃圾回收器負責回收無用對象佔據的內存資源

  • 垃圾回收器只知道釋放經由 new 分配的內存

  • 一旦垃圾回收器準備好釋放對象占用的存儲空間，將首先調用其 finalize() 方法，並在下一次垃圾回收動作發生時，才會真正回收對象佔用的內存

  • 垃圾回收器能夠提高對象創建的速度

    • 在 Java 虛擬機中，堆的實現像一個傳送帶，每分配一個新對象，他就往前跑一格，簡單地移動到尚未分配的區域(造成內存消耗)
    • 垃圾回收器將一面回收空間，一面使堆中的對象緊湊排列，使”堆指針”很容易移動倒更靠近傳送帶的開始處

  • Java 虛擬機將採用自適應的垃圾回收技術

    • 停止 - 複製(stop-and-copy)

      • 先暫停程序的運行，然後將所有存活的對象從當前堆複製到另一個堆，沒有被複製的全部屬於垃圾
      • 當對象被複製到新堆時，是一個挨著一個的，所以新堆保持緊湊排列
      • 把對象從一處搬到另一處時，所有指向他的那些引用都必須修正，位於堆或靜態存儲區的引用可以直接被修正，但可能還有其他指向這些對象的引用，在遍歷過程中才能被找到

    • 標記 - 清掃(mark-and-sweep)

      • 複製式回收器會將所有內存自一處複製到另一處，這樣相當浪費

        為了避免這種狀況，要是沒有新垃圾產生，就會轉換(自適應)到另一種工作模式 – 標記 - 清掃

      • 從堆棧和靜態存儲區出發，遍歷所有引用，找出存活的對象

        當找到存活對象，就會給對象設一個標記(這個過程還不會回收對象)，全部標記完成後，清掃工作才會開始

        沒有標記的對象將被釋放，不會發生任何複製動作(所以剩下的堆空間不連續，想得到連續空間就得重新整理剩下的對象)

  • Java 虛擬機中，內存分配以較大的”塊“為單位，如果對像較大，他會佔據單獨的塊

    “停止 - 複製”方法要求在釋放舊有對象之前，必須先把所有對象從舊堆複製到新堆(大量的複製行為)

    有了塊之後，垃圾回收器在回收的時候舊可以往廢棄的塊裡拷貝對象

    每個塊都有相應的代數(generation count)紀錄對象是否存活，如果塊被引用，代數會增加

    垃圾回收器會對上次回收動作之後新分配的塊進行清理

    此方法對小型短命的臨時對象十分有效，大型對象不會被複製(代數會增加), 內含小型對象的塊會被複製並整理

  • Java 的垃圾回收器: “自適應的、分代的、停止 - 複製、標記 - 清掃“

對象創建過程

  1. 即使沒有顯示使用 static, 構造器也是靜態方法

    因此，當首次創建類型為 Dog 的對象時，或者 Dog 類的靜態方法/靜態域首次被訪問時，Java 解釋器必須查找類路徑，定位 Dog.class 文件

  2. 載入 Dog.class (這將創案一個 Class 對象), 有關靜態初始化的所有動作都會執行

    因此，靜態初始化只在 Class 對象首次加載的時候進行一次

  3. 當用 new Dog() 創建對象的時候，首先將在堆上為 Dog 對象分配足夠的存儲空間

  4. 這塊存儲空間會被清零，自動將 Dog 對象中的所有基本類型數據都設置成默認值, 而引用設置為 null

  5. 執行所有出現於字段定義處的初始化動作

  6. 執行構造器

可變參數列表

package pers.jack.example.initialize;

public class OptionalTrailingArguments {
    static void f(int required, String... trailing) {
        System.out.print("required: " + required + " ");
        for (String s : trailing) {
            System.out.print(s + " ");
        }
        System.out.println();
    }

    public static void main(String[] args) {
        f(1, "one");
        f(2, "three");
        f(0);
    }
}
/*
required: 1 one 
required: 2 three 
required: 0 
*/

枚舉類型

  • enum 關鍵字使我們在需要群組並使用枚舉類型集時，可以很方便的處理
package pers.jack.example.initialize;

enum Spiciness {
    NOT, MILD, MEDIUM, HOT, FLAMING
}

public class SimpleEnumUse {
    public static void main(String[] args) {
        Spiciness howHot = Spiciness.HOT;
        System.out.println(howHot);
    }
}
/*
HOT
*/
  • enum 確實是類，且有一個特別實用的特性，其可以在 switch 語句內使用
package pers.jack.example.initialize;

public class Burrito {
    Spiciness degree;
    public Burrito(Spiciness degree) {
        this.degree = degree;
    }

    public void describe() {
        System.out.print("This burrito is ");
        switch (degree) {
            case NOT:
                System.out.println("not spicy at all");
                break;
            case MILD:
            case MEDIUM:
                System.out.println("a little hot");
                break;
            case HOT:
            case FLAMING:
            default:
                System.out.println("to hot");
        }
    }

    public static void main(String[] args) {
        Burrito
            plain = new Burrito(Spiciness.NOT),
            greenChile = new Burrito(Spiciness.MEDIUM),
            jalapeno = new Burrito(Spiciness.HOT);
        plain.describe();
        greenChile.describe();
        jalapeno.describe();
    }
}
/*
This burrito is not spicy at all
This burrito is a little hot
This burrito is to hot
*/
