Thinking in Java - 訪問權限控制

定制工具庫

定制 Print 方法以方便進行打印

package pers.jack.util
import java.io.*;

public class Print {
    //Print with a newline
    public static void println(Object obj) {
        System.out.println(obj);
    }

    //Print a newline by itsself
    public static void println() {
        System.out.println();
    }

    //Print with no line break
    public static void print(Object obj) {
        System.out.print(obj);
    }

    public static PrintStream printf(String format, Object... args) {
        return System.out.printf(format, args);
    }
}

Java 訪問權限修飾詞

控制對成員訪問權限的原因

  • 使用戶不要觸碰那些他們不該碰觸的部分
  • 為了讓類庫設計者可以更改類的內部工作方式，而不必擔心這樣會對客戶端程序員產生重大影響

默認: 包訪問權限

默認訪問權限沒有關鍵字，通常是指包訪問權限，意味當前包中的所有其他類對那個成員都有訪問權限

對於包之外的所有類，這個成員等同於 private

public: 所有人都可以訪問

public 之後緊跟著的成員聲明自己對每個人都是可用的

private: 你無法訪問

除了包含該成員的類之外，其他任何類都無法訪問這個成員

protected: 繼承訪問權限

protected 主要可以開放繼承的成員訪問，同時也提供包訪問權限，相同包內的其他類也可以訪問 protected 元素

總結

作用域 當前類 同 package 普通類 其他 package 普通類 同 package 派生類 其他 package 派生類
public √ √ √ √ √
protected √ √ x √ √
默認 √ √ x √ x
private √ x x x x

訪問權限的控制常被稱為是具體實現的隱藏，把數據和方法包裝進類中，以及具體實現的隱藏，常被共同稱作是封裝

類的訪問權限

  • 每個編譯單元(文件)都只能有一個 public 類

  • public 類的營稱必須完全與含有該編譯單元的文件名相匹配，包刮大小寫

  • 在創建一個包訪問權限(默認權限)，將該類的域聲明為 private 時才有意義 – 應盡可能地總是將域指定為私有的

  • 類既不可以是 private 的(這樣會使得除該類之外，其他任何類都不可以訪問它)，也不可以是 protected 的(內部類除外)

    如果不希望其他人對該類擁有訪問權限，可以把所有構造器(Constructor)指定為 private 的，從而阻止任何人創建該類的對象

    但是有例外，在該類的 static 成員內部可以創建

    Example

    class Soup1 {
    private Soup1() {}
    //Allow creation via static method
    public static Soup1 makeSoup() {
        return new Soup1();
    }
}

class Soup2 {
    private Soup2() {}
    /*
     * Create a static object and return a reference 
     * upon request (The "Singletom" Pattern 單例模式
     */
    private static Soup2 ps1 = new Soup2();
    public static Soup2 access() {
        return ps1;
    }
    public void f() {}
}

    單例模式始終只能創建它的一個對象。Soup2 類的對象是作為 Soup2 的一個 static private 成員而創建的，所以有且僅有一個
