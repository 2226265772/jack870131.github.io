Thinking in Java - 複用類

初始化

編譯器並不是簡單地位每一個引用都創建默認對象，如果想初始化這些引用，可以在下列位置進行:

  • 在定義對象的地方(意味著他們總是能夠在構造器被調用之前被初始化)
  • 在類的構造器中
  • 正要使用這些對象之前，此方式稱為惰性初始化(在生成對象不值得及不必每次都生成對象的情況下，這種方式可以減少額外負擔)
  • 使用實例初始化

Example

class Soap {
    private String s;
    //Constructor initialization
    Soap() {
        System.out.println("Soap()");
        s = "Constructed";
    }
    public String toString() {}
}

public class Bath {
    private String //Initailizing at point of definition
        s1 = "Happy",
        s2 = "Happy",
        s3, s4;
    private Soap castille;
    private int i;
    private float toy;
    public Bath() {
        System.out.println("Inside Bath()");
        s3 = "Jack";
        toy = 3.14f;
        castille = new Soap();
    }
    //Instance initializatino
    {i = 47;}
    public String toString() {
        if(s4 == null) //Delayed initialization
            s4 = "Jack";
        return
            "s1 = " + s1 + "\n" + 
            "s2 = " + s2 + "\n" +
            "s3 = " + s3 + "\n" +
            "s4 = " + s4 + "\n" +
            "i = " + i + "\n" +
            "toy = " + toy + "\n" +
            "castille = " + castille;
    }

    public static void main(String[] args) {
        Bath b = new Bath();
        System.out.println(b);
    }
}

繼承

  • 為了繼承，一班的規則是將所有的數據成員指定為 private, 所有的方法指定為 public (protected 也行)
  • 當創建一個導出類的對象時，該對象包含了一個基類的子對象
  • 基類對象的初始化: 構造器中調用基類構造器來執行初始化，而基類構造具有執行基類初始化所需要的所有知識和能力
  • Java 會自動在導出類的構造器中插入隊基類構造器的調用
  • 如果用一個帶參數的基類構造器，就必須用關鍵字 super 編寫調用基類構造器的子句
class Game {
    Game(int i) {
        System.out.println("Game constructor");
    }
}

class BoardGame extends Game {
    BoardGame(int i) {
        super(i);
        System.out.println("BoardGame constructor");
    }
}

public class Chess extends BoardGame {
    Chess() {
        super(11);
        System.out.println("Chess constructor");
    }
}

代理

將一個成員對象置於所要創造的類中(就像組合)，但與此同時我們在新類中暴露了該成員對象的所有方法(就像繼承)

Example

public class SpaceShipControls {
    void up(int velocity) {}
    void down(int velocity) {}
    void left(int velocity) {}
    void right(int velocity) {}
    void forward(int velocity) {}
    void back(int velocity) {}
    void turboBoost() {}
}
//代理
public class SpaceShipProxy {
    private String name;
    private SpaceShipControls controls = 
        new SpaceShipContrils();
    public SpaceShipProxy(String name) {
        this.name = name;
    }

    public void back(int velocity) {
        controls.back(velocity);
    }

    public void down(int velocity) {
        controls.down(velocity);
    }
    //...
}

組合與繼承之間選擇

  • 組合和繼承都允許在新的類中放置子對象(組合式顯示，繼承則是隱式的這樣做)
  • 組合通常用於想在新類中使用現有類的功能而非他的接口，即在新類嵌入某個對象，讓其實現所需要的功能

protected 關鍵字

  • 就類用戶而言，這是 private 的
  • 對於任何繼承於此類的導出類或其他任何位於通一個包內的類來說，他是可以訪問的(protected 也提供了包訪問權限)

向上轉型

  • 繼承技術重要的另一方面，除了”為新的類提供方法“，還有”新類是現有類的一種類型“

  • 向上轉型

    class Instrument {
    public void play() {}
    static void tune(Instrument i) {
        i.play()
    }
}

public class Wind extends Instrument {
    public static void main(String[] args) {
        Wind flute = new Wind();
        Instrument.tune(flute);
    }
}

final 關鍵字

  • final 指的是”這是無法改變的”

final 數據

  • 一個永不改變的編譯時常量

  • 一個在運行時被初始化的值，而你不希望他被改變

  • 一個既是 static 又是 final 的域只佔據一段不能改變的存儲空間

  • 對於基本類型，final 使數值恆定不便

    對於對象引用，final 使引用恆定不便

final 參數

  • 意味著你無法在方法中更改參數引用所指向的對象

final 方法

  • 使用 final 方法的原因有兩個
    • 把方法鎖定，以防任何繼承類修改他的涵義
    • 效率，如果將一個方法指明為 final, 就是同意編譯器將針對該方法的所有調用都轉為內嵌調用(在後期的 Java 版本中已經不需要使用 final 來進行優化了)

final 和 private 關鍵字

  • 類中所有的 private 方法都隱式地指定為是 final 的
  • 由於無法取用 private 方法，所以也就無法覆蓋

final 類

  • 表明你不打算繼承該類，也不允許他人這樣做
  • final 類禁止繼承，所以 final 類中所有的方法都隱式地指定為 final 的

初始化及類的加載

  • 一般來說，類的代碼在初次使用時才加載，通常是指類加載發生於創建類的第一個對象時
  • 當方穩 static 域或 static 方法時，也會發生加載
  • 所有的 static 對象和 static 代碼段都會在加載時依程序中的順序(定義類的書寫順序)依次初始化

Example

class Insect {
    private int i = 9;
    protected int j;
    Insect() {
        System.out.println("i = " + i + ", j = " + j); //4
        j = 39;
    }
    private static int x1 = 
        printInit("static Insect.x1 initialized"); //1
    static int printInit(String s) {
        System.out.println(s);
        return 47;
    }
}

public class Beetle extends Insect {
    private int k = printInit("Beetle.k initialized"); //5
    public Beetle() {
        System.out.println("k = " + k); //6
        System.out.println("j = " + j); 
    }
    private static int x2 = 
        printInit"static Beetle.x2 initialized"); //2
    public static void main(String[] args) {
        System.out.println("Beetle constructor"); //3
        Beetle b = new Beetle();
    }
}
/*
Output:
static Insect.x1 initialized
static Insect.x2 initialized
Beetle constructor
i = 9, j = 0
Beetle.k initialized
k = 47
j = 39
*/

總結

  • 繼承和組合都能從現有類型生成新類型。組合一般是將現有類型作為新類型底層實現的一部分來加以複用，而繼承父用的是接口
  • 一般應該優先選擇使用組合(或者代理)，只在確實必要時才使用繼承
