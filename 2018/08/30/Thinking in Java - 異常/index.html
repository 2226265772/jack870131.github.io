Thinking in Java - 異常

基本異常

  • 異常情形(exceptional condition)是指阻止當前方法或作用域繼續執行的問題

  • 拋出異常後，同 Java 中其他對象的創建一樣，將使用 new 在堆上創建異常對象。

    然當前的執行路徑被終止，並且從當前環境中彈出對異常對象的引用。

    異常處理機制接管程序，並開始尋找恰當的地方繼續執行程序(異常處理程序)

  • “拋出”一個異常:

    if (t == null) {
    throw new NUllPointerException();
}

  • 異常可以使得我們可以將每件事都當作一個事務來考慮，事務相當於計算機中的合同法，如果出了問題，只需要放棄整個計算

異常參數

  • 標準異常類都有兩個構造器: 一個是默認構造器，另一個是接受字符串作為參數，以便能把相關信息放入異常對象的構造器

    throw new NullPointerException("t = null");

  • 在使用 new 創建了異常對象之後，此對象的引用將傳給 throw

  • 能夠拋出任意類型的 Throwable 對象，他是異常類型的根類

捕獲異常

try 塊

  • 在這個塊裡嘗試各種方法調用

    try {
    // Code that might generate exceptions
}

異常處理程序

  • 緊跟在 try 塊之後，以關鍵字 catch 表示:

    try {
    // Code that might generate exceptions
} catch(Type1 id1) {
    // Handle exceptions of Type1
} catch(Type2 id2) {
    // Handle exceptions of Type2
} catch(Type3 id3) {
    // Handle exceptions of Type3
}

捕獲所有異常

  • 可以通過捕獲異常類型的基類 Exception 來做到這一點

    try {
    throw new Exception("My Exception");
} catch(Exception e) {
    System.out.println("Caught Exception");
}

棧軌跡

  • printStackTrace() 方法將返回一個由棧軌跡中的元素所構成的數組，其中每一個元素都表示棧中的一幀

  • printStackTrace() 方法顯示的將是原來異常拋出點的調用棧信息，並非重新拋出點的信息，想要更新這個信息，可以調用 flllInStackTrace() 方法，浙江返回一個 Throwable 對象，他是通過把當前調用棧信息填入原來那個異常對象而建立的

    catch (Exception e) {
    e.printStackTrace();
}

重新拋出異常

  • 有時希望把剛捕獲的異常重新拋出，尤其是在使用 Exception 捕獲所有異常的時候

    catch (Exception e) {
    System.out.println("An exception was thrown");
    throw e;
}

  • 重拋異常會把異常拋給上一級環境中的異常處理程序，同一個 try 塊的後續 catch 子句將被忽略

Example

public class Rethrowing {
    public static void f() throws Exception {
        System.out.println("originating the exception in f()");
        throw new Exception("thron from f()");
    }

    public static void g() throws Exception {
        try {
            f();
        } catch(Exception e) {
            System.out.println("Inside g(), e.printStackTrace()");
            e.printStackTrace(System.out);
            throw e;
        }
    }

    public static void h() throws Exception {
        try {
            f();
        } catch(Exception e) {
            System.out.println("Inside h(), e.printStackTrace()");
            e.printStackTrace(System.out);
            throw (Exception) e.fillInStackTrace();
        }
    }

    public static void main(String[] args) {
        try {
            g();
        } catch(Exception e) {
            System.out.println("main: printStackTrace()");
            e.printStackTrace(System.out);
        }
        try {
            h();
        } catch(Exception e) {
            System.out.println("main: printStackTrace()");
            e.printStackTrace(System.out);
        }
    }
}

Java 標準異常

  • Throwable 這個 Java 類被用來表示人和可以作為異常被拋出的類，Trowable 對象可以被分為兩種類型(從 Throwable 繼承):
    • Error: 用來表示編譯時和系統錯誤(一般可以不用關心)
    • Exception: 是可以被拋出的基本類型，在 Java 類庫，用戶方法以及運行故障中都可能拋出 Exception 異常

RuntimeException

  • 屬於運行時異的類型很多，他們會自動被 Java 虛擬基拋出，所以不必在異常說明中把他們列出來，這些異常都是從 RuntimeException 類型的異常
  • RuntimeException 類型的異常也被稱為”不受檢查異常“，這種異常將被自動捕獲，不必自己去檢查
  • 注意: 只能在代碼中忽略 RuntimeException (及其子類)類型的異常，其他類型異常的處理都是由編譯器強制實施的
  • RuntimeException 代表的是編程錯誤:
    • 無法預料的錯誤，比如從控制範圍之外傳遞進來的 null 引用

使用 finally 進行清理

  • 無論 try 塊中的異常是否拋出，他都能得到執行，這通常是用於內存回收之外的情況因為回收由垃圾回收器完成
  • finally 能保證: 無論 try 塊裡發生了什麼，內存總能得到釋放，但 Java 有垃圾回收機制，所以內存釋放不再是問題
  • 當要把廚內存之外的資源回復到他們的初始狀態時，就要用到 finally 子句

Example

class ThreeException extends Exception {}

public class FinallyWorks {
    static int count = 0;
    while (true) {
        try {
            if (count++ == 0) {
                throw new ThreeException();
            }
            System.out.println("No exception");
        } catch(ThreeException e) {
            System.out.println("ThreeException");
        } finally {
            System.out.println("In finally clause");
            if (count == 2) break;
        }
    }
}

把”被檢查的異常”轉換為”不檢查的異常”

  • 可以直接把”被檢查的異常”包裝進RuntimeException裡面:
try {
    // ... to do something useful
} catch(IDontKnowWhatToDoWithThisCheckedException e) {
    throw new RuntimeException();
}

異常使用指南

  • 應該在以下情況使用異常:
    • 在恰當的級別處理問題(在知道該如何處理得情況下才捕獲異常)
    • 解決問題並且重新調用產生異常的方法
    • 進行少許修補，然後繞過異常發生的地方繼續執行
    • 用別的數據進行計算，以代替方法預計會返回的職
    • 把當前雲型環境下能做的事情盡量做完，然後把相同的異常重拋到更高層
    • 把當前運行環境下能做的事情盡量做完，然後把不同的異常拋到更高層
    • 終止程序
    • 進行簡化(如果異常模式使問題變得太複雜，那用起來會非常煩人)
    • 讓類庫和程序更安全(這既是在為調適做短期投資，也是在為程序的健壯型做長期投資)
