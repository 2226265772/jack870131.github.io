Thinking in Java - 學習筆記(一)

最近在開始在讀 《Thinking in Java》 , 由於之前已經先上過畢向東的35天Java教程(筆記部分可參考 https://github.com/jack870131/java-note)和讀過《Head First Java》，所以這次比起紀錄語法方面的使用方法，我更傾向於記錄自己以前不太注意到的細節，也就是一些原理性的東西，比起知道 How, 我更想知道 Why。

數據存儲

程序運行時，對象是怎麼進行放置安排的呢? 有五個不同的地方可以進行數據的存儲。

  • 寄存器
    • 最快的存儲區
    • 位於處理器內部
    • 不能直接控制
    • C & C++ 允許向編譯器建議寄存器的分配方式
  • 堆棧 (Stack)
    • 位於 RAM (隨機訪問存儲器)
    • 通過堆棧指針可以從處理器獲得直接支持
    • 堆棧指針向下移動，分配新的內存，向上則釋放
    • Java 系統必須知道存儲在堆棧內所有項的生命週期
    • Java 對象引用存儲於堆棧中，但 Java 對象並不存儲於其中
  • 堆 (Heap)
    • 通用的內存池 (位於 RAM 區)
    • 存放所有 Java 對象
    • 編譯器不需要知道存儲的數據在堆裡存活多長時間
    • new 一個對象時，會自動在堆裡進存儲分配
    • 用堆進行存儲分配和清理可能比用堆棧進行存儲分配需要更多時間
  • 常量存儲
    • 常量值通常直接存放在程序代碼內部，永遠不會被外部改變
    • 存放在 ROM (只讀存儲器) 中
  • 非 RAM 存儲
    • 數據存活於程序之外的話，不受程序控制，程序沒有運行時也可以存在
    • 流對象: 對象轉化成字節流，通常被發送給另一台機器
    • 持久化對象: 對象被存放於磁盤上，程序終止仍可保持自己的狀態

static 關鍵字

  • 一般執行 new 來創建對象時，數據存儲空間才被分配，其方法才可供外界調用
  • 有兩種情形是用上述情形無法解決的:
    • 只想為某特定域分配單一存儲空間，不去考慮要創建多少對象
    • 希望某一個方法不予包含他的那個類的任何對象實例相關聯
  • 以下代碼 t1 & t2 對象共享同一個 i, t1.i 和 t2.i 指向同一存儲空間，因此他們具有相同的值50
public class StaticTest {
    public static int i = 50;
}

public static void main(String[] args) {
    StaticTest t1 = new StaticTest();
    StaticTest t2 = new StaticTest();
}
  • 靜態方法既可以像其他方法一樣，通過一個對象來飲用某個靜態方法，也可以通過 ClassName.method() 的形式加以引用
  • static 方法最重要的就是可以在不創建任何對象的前提下調用他
  • static 方法常拿來做”牧羊人”的角色，負責看護與其隸屬通一類型的實例群

常用標籤

  • @see: 引用其他類。此標籤允許用戶引用其他類的文檔，javadoc 會通過此標籤鏈接到其他文檔並加入一個具有超鏈接的“See Also” (參見) 的條目。格式如下:

    @see classname
@see fully-qualified-classname
@see fully-qualified-classname#method-name

  • @version: 包含在版本說明中的重要信息。格式如下:

    @version version-information (4.0)

  • @param: 該標籤用於方法文檔中，”parameter-name” 是方法的參數列表中的標示符。格式如下:

    @param parameter-name description

  • @return: 該標籤用於方法文檔，”description” 用來描述返回值的含義。格式如下

    @return description

  • @author: 作者的姓名或是帳號之類的訊息

  • @throws: 由於某個方法調用失敗而拋出的異常對象
