Thinking in Java - 控制執行流程

while & do-while 的區別

  • do-while 中的語句至少會執行一次，即便表達式第一次就被計算為 false
  • while 循環中，如果條件第一次就為 false, 其中的語句便不會被執行，while 較 do-while 常用

return, break, continue

return

  • 指定一個方法返回甚麼值(假設沒有 void 返回值)
  • 導致當前的方法退出，並返回指定值
  • 再返回 void 的方法中沒有顯式的 return 語句，但在該方法的結尾處有一個隱式的 return (可不必寫出)

break

  • 強行退出循環
  • 不執行循環中剩餘的語句

continue

  • 停止執行當前的迭代
  • 退回循環起始處，開始下一次迭代

規則

  • 一般 continue 會退回最內層循環的開頭(頂部)，並繼續執行
  • 帶標籤的 continue 會到達標籤的位置，隨後重新進入緊接在那個標籤後面的循環
  • 一般的 break 會中斷並跳出當前循環
  • 帶標籤的 break 會中斷並跳出標籤所指的循環

Example

label;
outer-iteration {
    inner-iteration {
        ...
        break; //(1)
        ...
        continue; //(2)
        ...
        continue label; //(3)
        ...
        break; //(4)
    }
}
  • (1) break 中斷內部迭代，回到外部迭代
  • (2) continue 使執行點移回內部迭代的起始處
  • (3) continue label 同時中斷內部迭代以及外部迭代，直接轉到 label 處; 繼續迭代，但是是從外部迭代開始
  • (4) break label 中斷所有迭代，並回到 label 處，但並不重新迭代
