Thinking in Java - 多態

介紹

介紹

  • 多態通過分離做甚麼和怎麼做，從另一角度將接口和實現分離

  • 封裝通過合併特徵和行為來創建新的數據類型

    實現隱藏則通過將細節私有化把接口和實現分離開來

    多態的作用是消除類型之間的耦合關係

向上轉型

  • 對象既可以做為他自己本身的類型使用，也可以作為他的基類型使用
  • 導出類的接口不能被基類方問，一旦向上轉型，既不能調用導出類的新方法

缺陷

  • “覆蓋”私有方法

    public class PrivateOverride {
    private void f() {System.out.println("private f()");}
    public static void main(String[] args) {
        PrivateOverride po = new Derived();
        po.f();
    }
}

class Derived extends PrivateOverride {
    public void f() {
        System.out.println("public f()");
    }
}
/*
Output:
private f()
*/

    我們希望輸出 public f(), 但由於 private 方法被自動認為是 final 方法，而且對導出類是屏蔽的。因此在 Derived 類中的 f() 方法是全新的方法

  • 域與靜態方法

    class Super {
    public int field = 0;
    public int getField() {return field;}
}

class Sub extends Super {
    public int field = 1;
    public int getField() {return field;}
    public int getSuperField() {return super.field;}
}

public class FieldAccess {
    public static void main(String[] args) {
        Super sup = new sub(); //Upcast
        System.out.println("sup.field = " + sup.getField());
        Sub sub = new Sub();
        System.out.println("sub.field = " +
                          sub.getField() + 
                           ", sub.getSuperField() = " +
                           sub.getSuperfield());
    }
}
/*
Output:
sup.field = 0, sup.getField = 1
sub.field = 1, sub.getField() = 1, sub.getSuperField() = 0
*/

    當 Sub 對象轉型為 Super 引用時，任何域訪問操作都將由編譯器解析，因此不是多態的

    但通常會將所有的域設為 private, 因此不能直接訪問

    如果某個方法是靜態的，他的行為就不具多態性

    靜態方法是與類，並非單個對象相關聯的

    class StaticSuper {
    public static String staticGet() {
        return "Base dynamicGet()";
    }
    public String dynamicGet() {
        return "Base dynamicGet()";
    }
}

class StaticSub extends StaticSuper {
    public static String staticGet() {
        return "Derived staticGet()";
    }
    public String dynamicGet() {
        return "Derived dynamicGet()";
    }
}

public class StaticPolymorphism {
    public static void main(String[] args) {
        StaticSuper sup = new StaticSub(); //Upcast
        System.out.println(sup.staticGet());
        System.out.println(sup.dynamicGet());
    }
}
/*
Output:
Base staticGet()
Derived dynamicGet()
*/

    構造器和多態

    • 構造器的任務: 檢查對象是否被正確的構造，必須令所有構造器都得到調用，否則就不可能正確構造完整對象
    class Meal {
    Meal() {System.out.println("Meal()");} //1
}
class Bread {
    Bread() {System.out.println("Bread()");} //5
}
class Cheese {
    Cheese() {System.out.println("Cheese()");} //4
}
class Lettuce {
    Lettuce() {System.out.println("Lettuce()");} //6
}
class Lunch extends Meal {
    Lunch() {System.out.println("Lunch()");} //2
}
class PortableLunch extends Lunch { 
    PortableLunch() {System.out.println("PortableLunch()");} //3
}
public class Sandwich extends PortableLunch {
    private Bread b = new Bread();
    private Cheese c = new Cheese();
    private Lettuce l = new Lettuce();
    public Sandwich() {System.out.println("Sandwich()");} //7
    public static void main(String[] args) {
        new Sandwich();
    }
}
/*
Meal()
Lunch()
PortableLunch()
Bread()
Cheese()
Lettuce()
Sandwich()
*/

    調用順序

    • 調用基類構造器，這個步驟會不斷反覆地規，首先是構造這種層次結構的根，然後是下一層導出類，直到最底層的導出類
    • 按聲明順序調用成員的初始化方法
    • 調用導出類構造器的主體

    繼承與清理

    • 子對象通常都會留給垃圾回收器進行清理
    • 某個子對象要依賴於其他對象，銷毀的順序應該和初始化順序相反
    • 對於字段，則意味著銷毀與聲明的順序相反(字段的初始化是按照聲明的順序進行的
    • 對於基類，應該首先對其導出類進行清理，然後才是基類

    用繼承進行設計

    • 優先選擇組合，組合較繼承更為靈活，因為它可以動態選擇類型(因此也就選擇了行為)

      相反，繼承在編譯時就需要知道確切類型

      Example

      class Actor {
    public void act() {}
}
class HappyActor extends Actor {
    public void act() {System.out.println("HappyActor");}
}
class SadActor extends Actor {
    public void act() {System.out.println("SadActor");}
}
class Stage {
    private Actor actor = new HappyActor();
    public void change() {actor = new SadActor();}
    public void performPlay() {actor.act();}
}
public class Transmogrify {
    public static void main(String[] args) {
        Stage stage = new Stage();
        stage.performPlay();
        stage.change();
        stage.performPlay();
    }
}
/*
Output:
HappyActor
SadActor
*/

  • 通用準則: 用繼承表達行為間的差異，並用字段表達狀態上的變化

  • 繼承中導出類與基類的關系是“is=like-a(像一個)”的關係，因為導出類就像是一個基類 – 他有著相同的基本接口，但是他還具有額外方法實現的其他特性
