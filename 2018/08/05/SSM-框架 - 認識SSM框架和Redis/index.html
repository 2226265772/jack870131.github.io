SSM框架 - 認識SSM框架和Redis

最近開始進入學習 Java 框架的部分了，閱讀的是《JavaEE 互聯網輕量級框架整合開發 – SSM框架(Spring + SpringMVC + MyBatis)和 Redis 實現》 這本書。一樣將我的每次閱讀的學習心得放上這裡跟大家分享。

Spring 框架

Spring IoC (Inversion of Control, 控制反轉)

  • IoC 是一個容器，他會認為依竊 Java 資源都是 Java Bean, 容器的目標為管理這些 Bean 和他們之間的關係

  • 個個 Java Bean 之間會存在一定的依賴關係，比如班級是一來於老師和學生組成，兩者之間形成了依賴關係，老師和學生有教育和被教育的關係

  • Spring 是依靠描述來完成對象的創建及其依賴關係的

  • 如果使用者想要使用插座A，我們可以通過代碼來實現

    Socket socket = new SocketA();
user.setSocket(socket);
user.useSocket();

    我們會發現Socket 接口和實現類 SocketA 耦合了，當用戶要使用插座B時，就要修改代碼

    Socket socket = new SocketB();
user.setSocket(socket);
user.useSocket();

    Spring IoC 可以解決這個問題，用戶不須通過 new 的方式創建對象，而是使用配置的方式讓 Spring IoC 容器通過配置找到插座

    <bean id="socket" class="Socket1"/>
<bean id="user" class="xxx.User">
    <property name="socket" ref="socket"/>
</bean>

    如果用戶需要插座B則只需要將配置切換為

    <bean id="socket" class="Socket2">

    我們可以發現 Socket 接口不再依賴於某個實現類，需要實現類時通過配置信息就可以完成，不需要再重新 new 創建對象了

Spring AOP (Aspect Oriented Programming, 面向切面編程)

  • 切面的意義: 舉個例子，如果一個訂單要通過生產部門和財務部門的審批，第一關生產部門審核通過後，在財務部門發現預算超支要駁回，此決定影響的不僅是財務部門，更影響了之前所做的審批。那麼預算超支就是所謂的切面
  • Spring AOP 常用於數據庫事務編程。如同上面的例子，只要出現異常消息，那麼 Spring 就會將事務回滾，使之前的數據更新也作廢，從而保證數據的一致性，不需要通過代碼來實現這個過程

MyBatis

  • MyBatis 的優勢在於靈活，幾乎可以代替 JDBC，同時提供了接口編程

  • MyBatis 的 DAO 層(數據訪問層)是不需要實現類的，他只需要一個街口和 XML (或者注解)

  • Hibernate 基本不需要編寫 SQL 就可以通過對象關係映射(Object Relational Mapping, ORM) 來操作數據庫

    MyBatis 則需編寫提供 SQL 去運行，程序員可自定義 SQL 規則，從而優化性能

  • 對於性能要求不高的系統，如管理系統、ERP 等可使用 Hibernate

    對於性能要求高、響應快的系統則可使用 MyBatis

Spring MVC

  • MVC 模式把應用程序(輸入邏輯、業務邏輯和 UI 邏輯)分成不同的方面

    • Model (模型)，封裝了應用程序的數據和由他們組成的 POJO
    • View (視圖)，負責把模型數據渲染到試圖上，將數據以一定的形式展現給用戶
    • Controller (控制器)，負責處理用戶請求，並建立適當的模型把它傳遞給視圖渲染

  • Spring MVC 還可以定義邏輯視圖，通過其提供的試圖解析器就能夠很方便的找到對應的視圖進行渲染

    可以使用其消息轉換的功能，比如在 Controller 的方法內加入注解 @ResponseBody, Spring MVC 就可以通過消息轉換，將數據轉換為 JSON, 提供給前端 Ajax 使用

Redis

  • Redis 是目前相當流行的 NoSQL (Not Only SQL)。它可以提高互聯網系統的性能，具備一定持久層的功能，也可以做為一種緩存工具

  • 對於常用數據，第一次從數據庫讀出，然後就存放在 NoSQL 中，這樣以後就無須再訪問數據庫，只需從 NoSQL 中讀出即可，性能就會比數據庫快

    高併發的操作，可以在 NoSQL 上完成寫入，等待某一時刻再批量寫入數據庫

  • 傳統數據庫需要讀磁盤進行數據讀取

    NoSQL 從數據庫中讀取數據進行緩存，就可以從內存中讀取數據，更能滿足高併發、高速處理數據得場合
