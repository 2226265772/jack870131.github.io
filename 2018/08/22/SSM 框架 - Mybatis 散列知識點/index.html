SSM 框架 - Mybatis 散列知識點

這邊先簡單且分散的記錄一點自己學習 Mybatis 的心得，主要由慕客網(https://www.imooc.com/learn/154)進行學習，之後自己融會貫通後再好好寫正式的 SSM 框架知識點整理

Mybatis 特點

  • SQL 語句與代碼分離

    • 優點: 便於管理和維護
    • 缺點: 不便於調試，需要借助日誌工具(log4j)獲得信息

  • 用標籤控制動態 SQL 的拼接

    • 優點: 用標籤代替編寫邏輯代碼
    • 缺點: 拼接複雜 SQL 語句時，沒有代碼靈活，比較複雜

  • 結果集與 Java 對象的自動映射

    • 優點: 暴政名稱相同即可自動映射
    • 缺點: 對開發人員所寫的 SQL 依賴性很強

  • 編寫原生 SQL

    • 優點: 接近 JDBC, 很靈活

    • 缺點: 對 SQL 語句依賴程度很高

DAO 層需求

  • 對象能與數據庫交互

Mybatis 之 SqlSession

  • SqlSession 作用:
    • 向 SQL 語句傳入參數
    • 執行 SQL 語句
    • 獲取執行 SQL 語句的結果
    • 事務的控制
  • 如何得到 SqlSession:
    • 通過配置文件獲取數據庫連接相關信息
    • 通過配置信息構建 SqlSessionFactory
    • 通過 SqlSessionFactory 打開數據庫會話(SqlSession)

OGNL 表達式

  • 取值範圍: 標籤的屬性中
  • 取值寫法(大小寫敏感):
    • String 與基本數據類型: _parameter
    • 自定義類型(Message): 屬性名(如: command)
    • 集合:
      • 數組: array
      • List: list
      • Map: _parameter
  • 操作符:
    • java 常用操作符: +, -, *, /, ==, !=, ||, && 等
    • 自己特有操作符: and, or, mod, in, not in
  • 從集合中取出一條數據
    • 數組:
      • array[索引](String[ ])
      • array[索引].屬性名(Message[ ])
    • List:
      • list[索引](List<String>)
      • list[索引].屬性名(List<Message>)
    • Map:
      • _parameter.key(Map<String, String>) -> _parameter 可省略
      • key.屬性名(Map<String, Message>)
  • 利用 foreach 標籤從集合中取出數據
    • <foreach collection=“array” index=“i” item=“item”>
      • 數組 -> i : 索引(下標), item : item
      • List -> i : 索引(下標), item : item
      • Map: -> i : key, item : item.屬性名

接口式編程

動態代理過程

IMessage.queryMessageList()

  1. MapperProxy implements InvocationHandler

    類裡實現個方法: MapperProxy.invoke()

  2. 創建代理實例:

    Proxy.newProxyInstance(類加載器、接口、MapperProxy 對象)

  3. 由 sqlSession.getMapper() == Proxy.newProxyInstance() 獲取

  4. 觸發 invoke() 方法:

    IMessage iMessage == Proxy.newProxyInstance

    iMessage.queryMessageList() == MapperProxy.invoke()

MapperProxy.invoke() == sqlSession.selectList()

  1. 加載配置信息(Mybatis 中相關的類: Configuration)

  2. 因為: 接口全名稱.方法名 == namespace.id

  3. 所以: MapperProxy.invoke() 中有這樣一行代碼:

    sqlSession.selectList(namespace.id, parameter)

IMessage iMessage = Proxy.newProxyInstance()

結論

iMessage.queryMessageList(parameter) = sqlSession.selectList(namespace.id, parameter)

  • 注意事項
    • 要攔截什麼樣的對象
    • 攔截對象什麼行為
    • 什麼時候攔截
